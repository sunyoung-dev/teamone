# TeamOne WB - 기획서 (Planning Document)

## 1. 서비스 개요

**TeamOne WB**는 22명으로 구성된 여성 야구팀의 경기 기록, 선수 통계, 라인업 관리를 위한 모바일 우선 웹 애플리케이션이다.

### 핵심 목표
- 경기 당일 현장에서 모바일로 타석별 기록을 즉시 입력
- 자동 계산된 선수별/팀별 통계를 언제든 확인
- 시즌 경기 결과와 팀 성적을 한눈에 파악
- 매 경기 라인업(타순, 포지션)을 편리하게 구성하고 저장

### 대상 사용자
| 역할 | 설명 | 주 사용 기능 |
|------|------|-------------|
| 기록원 | 경기 중 타석 결과 실시간 입력 | 경기 기록지, 라인업 |
| 감독/코치 | 라인업 구성, 통계 분석 | 라인업 관리, 선수 통계, 대시보드 |
| 선수 | 본인 및 팀 성적 확인 | 선수 통계, 팀 결과 |

### 기술 스택 요약
- **Frontend:** React + Vite + Material UI (MUI)
- **Backend:** Node.js + Express
- **Storage:** JSON 파일 기반 (서버 `data/` 디렉토리)
- **배포:** 단일 서버 (개발/소규모 운영)

## 2. 사용자 시나리오 (경기 당일 흐름)

### 시나리오: 리그 경기일

```
[경기 전]
1. 감독이 모바일에서 "라인업 관리" 진입
2. 오늘 경기의 상대팀, 날짜, 장소 입력
3. 22명 중 출전 선수 선택 → 타순 배정 → 포지션 배정
4. 라인업 확정 및 저장

[경기 중]
5. 기록원이 "경기 기록지" 진입 → 오늘 경기 선택
6. 이닝별로 각 타자의 타석 결과를 터치로 입력
   - 예: 1회초 1번 타자 → "1H" (안타) 선택
   - 예: 1회초 2번 타자 → "GO" (땅볼 아웃) 선택
7. 이닝 종료 시 다음 이닝으로 이동
8. 경기 종료 시 최종 스코어 확인 및 결과(승/패/무) 기록

[경기 후]
9. "선수 통계" 페이지에서 자동 업데이트된 타율, OBP, SLG 확인
10. "팀 대시보드"에서 시즌 성적 요약 확인
```

### 핵심 UX 원칙
- 경기 중 기록은 **한 손 조작** 가능해야 함 (큰 버튼, 단순 터치)
- 타석 결과 입력은 **2탭 이내** 완료 (타자 선택 → 결과 선택)
- 실수 입력 시 **즉시 수정** 가능
- 네트워크 불안정 시에도 **입력 데이터 유실 방지** (로컬 임시 저장 고려)

## 3. 핵심 기능 정의

### 3.1 경기 기록지 (Scorebook)

**목적:** 경기 중 타석별 결과를 실시간으로 기록

**기능 상세:**
- 이닝별 타석 기록 입력 (타자 순서대로)
- 타석 결과 코드 선택 (터치 기반 UI)
  - 안타류: `1H` (단타), `2H` (2루타), `3H` (3루타), `HR` (홈런)
  - 아웃류: `GO` (땅볼 아웃), `FO` (플라이 아웃), `SO` (삼진), `DP` (병살)
  - 출루류: `BB` (볼넷), `HBP` (몸에 맞는 공), `E` (실책 출루)
  - 희생류: `SF` (희생 플라이), `SH` (희생 번트)
- 입력한 기록 수정/삭제
- 이닝별 스코어 자동 집계
- 경기 완료 처리 (최종 스코어, 승패 확정)

**수용 기준:**
- 타석 결과를 2탭 이내로 입력 가능
- 이닝 전환이 직관적
- 기존 기록 수정이 가능

### 3.2 선수 통계 (Player Stats)

**목적:** 경기 기록 기반으로 선수별 통계를 자동 계산하여 표시

**계산 통계 항목:**
| 통계 | 계산식 | 설명 |
|------|--------|------|
| AVG (타율) | 안타 / 타수 | 타수 = 타석 - BB - HBP - SF - SH |
| OBP (출루율) | (안타+BB+HBP) / (타수+BB+HBP+SF) | |
| SLG (장타율) | 루타 / 타수 | 루타 = 1H*1 + 2H*2 + 3H*3 + HR*4 |
| OPS | OBP + SLG | |
| PA (타석) | 전체 타석 수 | |
| AB (타수) | PA - BB - HBP - SF - SH | |
| H (안타) | 1H + 2H + 3H + HR | |
| HR (홈런) | 홈런 수 | |
| RBI (타점) | 별도 입력 또는 향후 확장 | v1에서는 미포함 |
| BB (볼넷) | 볼넷 수 | |
| SO (삼진) | 삼진 수 | |

**기능 상세:**
- 선수 목록 (등번호, 이름, 포지션)
- 선수별 상세 통계 페이지
- 시즌 전체 통계 / 최근 N경기 통계 필터
- 통계 항목별 정렬 (타율순, 홈런순 등)

**수용 기준:**
- 경기 기록 입력 즉시 통계에 반영
- 소수점 3자리 표시 (타율 .333 형식)
- 선수 목록에서 주요 통계 한눈에 확인 가능

### 3.3 팀 경기 결과 (Team Results)

**목적:** 시즌 전체 경기 결과를 요약하여 표시

**기능 상세:**
- 경기 목록 (날짜, 상대팀, 스코어, 결과)
- 시즌 성적 요약 (승-패-무, 승률)
- 경기별 상세 보기 (기록지 연결)
- 월별/기간별 필터

**수용 기준:**
- 최신 경기가 상단에 표시
- 승/패/무가 색상으로 구분
- 경기 클릭 시 상세 기록 확인 가능

### 3.4 라인업 관리 (Lineup Management)

**목적:** 경기별 출전 라인업(타순, 포지션)을 구성하고 저장

**기능 상세:**
- 22명 선수 풀에서 출전 선수 선택
- 타순 (1-9번) 배정 (드래그 또는 터치로 순서 변경)
- 포지션 배정 (투수, 포수, 1루수, 2루수, 3루수, 유격수, 좌익수, 중견수, 우익수, DH)
- 라인업 저장 및 수정
- 이전 경기 라인업 복사 기능

**수용 기준:**
- 선수 선택과 타순 배정이 직관적
- 포지션 중복 배정 방지 (경고)
- 이전 라인업 불러오기 가능

### 3.5 팀 대시보드 (Team Dashboard)

**목적:** 팀 전체 현황을 한눈에 파악할 수 있는 메인 화면

**기능 상세:**
- 시즌 성적 요약 카드 (승-패-무, 승률)
- 최근 경기 결과 (최근 5경기)
- 팀 주요 통계 (팀 타율, 팀 OBP 등)
- 개인 통계 리더보드 (타율 TOP 5, 홈런 TOP 5 등)
- 다음 경기 일정 (등록된 경우)

**수용 기준:**
- 앱 진입 시 첫 화면으로 표시
- 주요 정보가 스크롤 없이 확인 가능 (모바일 기준)
- 각 카드에서 상세 페이지로 이동 가능

## 4. 데이터 모델 (JSON 구조)

### 4.1 players.json

선수 22명의 기본 정보를 저장한다.

```json
{
  "players": [
    {
      "id": "p001",
      "name": "김수영",
      "number": 1,
      "position": "SS",
      "battingHand": "R",
      "throwingHand": "R",
      "active": true
    }
  ]
}
```

**필드 설명:**
| 필드 | 타입 | 설명 | 예시 |
|------|------|------|------|
| id | string | 고유 ID (p + 3자리) | "p001" |
| name | string | 선수 이름 | "김수영" |
| number | number | 등번호 | 1 |
| position | string | 주 포지션 코드 | "SS", "P", "C", "1B", "2B", "3B", "LF", "CF", "RF", "DH" |
| battingHand | string | 타석 (L/R/S) | "R" |
| throwingHand | string | 투구 손 (L/R) | "R" |
| active | boolean | 활성 여부 | true |

### 4.2 games.json

경기 정보와 타석 기록을 저장한다.

```json
{
  "games": [
    {
      "id": "g001",
      "date": "2026-04-05",
      "opponent": "블루베리스",
      "venue": "서울 야구장",
      "result": "W",
      "scoreOurs": 5,
      "scoreTheirs": 3,
      "innings": 7,
      "status": "final",
      "lineup": [
        {
          "playerId": "p001",
          "battingOrder": 1,
          "position": "SS"
        },
        {
          "playerId": "p005",
          "battingOrder": 2,
          "position": "CF"
        }
      ],
      "atBats": [
        {
          "inning": 1,
          "playerId": "p001",
          "result": "1H",
          "order": 1
        },
        {
          "inning": 1,
          "playerId": "p005",
          "result": "GO",
          "order": 2
        }
      ]
    }
  ]
}
```

**games 필드 설명:**
| 필드 | 타입 | 설명 | 값 범위 |
|------|------|------|---------|
| id | string | 고유 ID (g + 3자리) | "g001" |
| date | string | 경기 날짜 (ISO) | "2026-04-05" |
| opponent | string | 상대팀 이름 | "블루베리스" |
| venue | string | 경기 장소 | "서울 야구장" |
| result | string | 경기 결과 | "W" / "L" / "D" / null (진행중) |
| scoreOurs | number | 우리 팀 득점 | 0 이상 |
| scoreTheirs | number | 상대 팀 득점 | 0 이상 |
| innings | number | 총 이닝 수 | 기본 7 |
| status | string | 경기 상태 | "scheduled" / "in_progress" / "final" |

**lineup 필드 설명:**
| 필드 | 타입 | 설명 |
|------|------|------|
| playerId | string | 선수 ID (players.json 참조) |
| battingOrder | number | 타순 (1-9) |
| position | string | 해당 경기 포지션 |

**atBats 필드 설명:**
| 필드 | 타입 | 설명 |
|------|------|------|
| inning | number | 이닝 번호 |
| playerId | string | 타자 선수 ID |
| result | string | 타석 결과 코드 |
| order | number | 해당 이닝 내 타석 순서 |

**타석 결과 코드 (At-Bat Result Codes):**
| 코드 | 영문 | 한글 | 분류 |
|------|------|------|------|
| 1H | Single | 단타 | 안타 (타수O, 출루O) |
| 2H | Double | 2루타 | 안타 (타수O, 출루O) |
| 3H | Triple | 3루타 | 안타 (타수O, 출루O) |
| HR | Home Run | 홈런 | 안타 (타수O, 출루O) |
| GO | Ground Out | 땅볼 아웃 | 아웃 (타수O) |
| FO | Fly Out | 플라이 아웃 | 아웃 (타수O) |
| SO | Strikeout | 삼진 | 아웃 (타수O) |
| DP | Double Play | 병살 | 아웃 (타수O) |
| BB | Walk | 볼넷 | 출루 (타수X) |
| HBP | Hit By Pitch | 몸에 맞는 공 | 출루 (타수X) |
| SF | Sacrifice Fly | 희생 플라이 | 아웃 (타수X) |
| SH | Sacrifice Bunt | 희생 번트 | 아웃 (타수X) |
| E | Error | 실책 출루 | 출루 (타수O) |

## 5. 페이지 목록 및 내비게이션 구조

### 5.1 페이지 목록

| # | 경로 | 페이지 명 | 설명 |
|---|------|----------|------|
| 1 | `/` | 팀 대시보드 | 메인 화면, 팀 현황 요약 |
| 2 | `/games` | 경기 목록 | 전체 경기 결과 리스트 |
| 3 | `/games/new` | 새 경기 등록 | 경기 정보 + 라인업 입력 |
| 4 | `/games/:id` | 경기 상세 | 경기 기록지 (타석 기록 입력/조회) |
| 5 | `/games/:id/lineup` | 라인업 편집 | 해당 경기 라인업 수정 |
| 6 | `/players` | 선수 목록 | 전체 선수 + 주요 통계 |
| 7 | `/players/:id` | 선수 상세 | 개인 상세 통계 |
| 8 | `/players/manage` | 선수 관리 | 선수 추가/수정/삭제 |

### 5.2 내비게이션 구조

```
Bottom Navigation Bar (모바일 하단 고정, 4탭)
├── 대시보드 (HomeIcon)        → /
├── 경기 (SportsBaseballIcon)  → /games
├── 선수 (PeopleIcon)          → /players
└── 기록 (EditNoteIcon)        → /games (최근 진행중 경기로 이동)
```

**내비게이션 원칙:**
- 하단 네비게이션 바는 항상 표시 (경기 기록 입력 중에도)
- "기록" 탭은 현재 진행 중인 경기가 있으면 해당 경기 기록지로 직접 이동
- 진행 중인 경기가 없으면 경기 목록으로 이동
- 상단 앱바에 뒤로가기, 페이지 제목 표시
- 깊은 depth는 최대 2단계 (목록 → 상세)

### 5.3 화면 흐름도

```
[대시보드] ──→ [경기 목록] ──→ [새 경기 등록] ──→ [라인업 편집] ──→ [경기 기록지]
    │               │                                                    │
    │               └──→ [경기 상세(기록지)]  ←──────────────────────────┘
    │
    └──→ [선수 목록] ──→ [선수 상세 통계]
              │
              └──→ [선수 관리]
```

## 6. 범위 및 우선순위

### v1 (MVP) 범위
1. 선수 등록/관리
2. 경기 생성 및 라인업 구성
3. 타석별 기록 입력
4. 자동 통계 계산 (AVG, OBP, SLG, OPS)
5. 팀 경기 결과 목록
6. 대시보드 (시즌 요약, 리더보드)

### v1 제외 (향후 확장)
- 투수 통계 (ERA, WHIP 등) - 투수 기록 모델 별도 필요
- 타점(RBI), 득점(R) 추적 - 주자 상황 추적 필요
- 도루 기록
- 수비 통계
- 사용자 인증/권한 관리
- 사진/영상 첨부
- 푸시 알림
